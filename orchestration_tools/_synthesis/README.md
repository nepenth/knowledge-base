## Technical Patterns

### 1. Declarative Orchestration

**Description**:  
A pattern where the desired state of a system is defined declaratively, and the orchestration tool ensures that the actual state converges to the desired state. This approach is commonly used in configuration management tools like Ansible.

**Implementation Notes**:  
- Define clear and consistent state definitions.  
- Handle idempotency to ensure repeatable operations.  
- Ensure robustness against failures in orchestration logic.
