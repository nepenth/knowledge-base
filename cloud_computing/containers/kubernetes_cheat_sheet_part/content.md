**List of Features**

* **NodePort Service**: Exposes a service on a static port on each node in a cluster.
* **Endpoints**: A list of IP addresses and ports that a service forwards traffic to.
* **Resource Quotas**: Limits the amount of resources (CPU, memory, etc.) a namespace can use.
* **LimitRange**: Defines resource usage limits for containers in a namespace.
* **Custom Resource Definition (CRD)**: Extends Kubernetes to manage custom resources.
* **Operator**: A custom controller that manages complex applications on Kubernetes.
* **Admission Controller**: Intercepts requests to the API server and allows or denies them based on configured rules.
* **Finalizer**: Ensures that specific cleanup steps are performed when a resource is deleted.
* **Horizontal Pod Autoscaler (HPA)**: Automatically scales the number of replicas in a deployment based on CPU utilization.
* **Vertical Pod Autoscaler (VPA)**: Automatically adjusts the resources allocated to pods based on their actual usage.
* **Cluster Autoscaler**: Automatically adds or removes nodes from a cluster based on resource utilization.
* **Self-healing**: Automatically replaces and reschedules failed containers.
* **Secrets Management**: Manages sensitive information like passwords and API keys securely.
* **Resource Quotas**: Limits the amount of resources (CPU, memory, etc.) that can be consumed by pods in a namespace.
* **Default Namespace**: The default namespace for newly created objects.
* **Worker Node**: A node that runs pods and is responsible for their execution.
* **Controller Manager**: Manages control plane components like nodes, services, and persistent volumes.
* **ETCD**: A distributed key-value store used to store configuration data and provide a single source of truth for Kubernetes clusters.
